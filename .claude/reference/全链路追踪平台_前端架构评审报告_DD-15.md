## 二、五维度详解

### 1. 技术栈健康度（20/50）
- **优**：锁定 TS、自研可控、图表库（ECharts/D3）认知正确。
- **伤**：系统设计说明书 §2.7 留了"Vue3 / React"二选一，**框架未决**即 UI 库/状态管理/路由/构建工具全悬空——阻断项，页面与 Mock 无法开工。
- **框架专项检查（二选一约束）**：
  - 选 React：须 **Hooks + 函数组件**；服务端状态用 TanStack Query，客户端轻全局用 Zustand；重型可视化 `memo`/`useMemo` 防重渲染。
  - 选 Vue3：须 **`<script setup>` + Composition API + Pinia**；严禁 Options API 混写。

### 2. 架构设计模式（23/50）
- 9 个组件清单合理，但**无目录结构、无分层、无路由表、无状态归属**。8 组页面（概览/拓扑/Trace/APM/基建/告警/管理/采集）必须 Feature-based 拆分，否则迅速上帝文件化。

### 3. 工程化成熟度（18/50）
- 完全空白：无 Vite 配置、无 TS strict、无 ESLint/Prettier/Husky、无单测（Vitest）/E2E（Playwright）、无 CI、无路由级代码分割（g6/echarts 重型库必分包，否则首屏崩）。

### 4. 性能与可维护性（25/50）
- 有 Canvas/缓存意识，但 4 真问题：①万级拓扑节点无 LOD/聚类/WebGL；②火焰图/瀑布从 5 万 Span 建树无 Web Worker 卸载；③实时 WS vs 轮询未决且无统一层；④客户端缓存未提（Query/SWR）。

### 5. 综合
- 属**决策缺失型短板**非设计错误，优化收益高。

## 四、关键优化落点（已写入 DD-15 v0.2）

1. **技术栈表**：给出 React 线 / Vue3 线双基线，框架二选一交前端负责人决策（P0-1）。
2. **架构分层**：`app / features / shared / lib / types` Feature-based 结构，业务逻辑下沉 hooks，服务端状态走 Query。
3. **性能专项**：万级拓扑 G6 WebGL + LOD + 聚类 + Worker 布局；火焰/瀑布 Worker 建树 + Canvas 虚拟化；viz 库路由分包 + Lighthouse CI。
4. **实时定稿**：WebSocket 统一层 + SSE/轮询降级 + 心跳重连 + Query 缓存，按页面分级。
5. **a11y/主题**：WCAG AA、色盲配色（色+图标+文字）、键盘导航、设计令牌双主题。
6. **原 4 待评审项全部给出推荐方案并消化**。
