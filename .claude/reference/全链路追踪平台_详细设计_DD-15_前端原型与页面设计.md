## 1. 技术栈选型（评审 P0-1，推荐方案）

> 系统设计说明书 §2.7 已定"自研 Vue3 / React + TypeScript"。**评审决议 D1：选用 Vue3 线**（公司前端团队更熟悉 Vue 技术栈），React 线作为备选。下表以 Vue3 为选定基线。

| 层 | 选定基线（Vue3 线，D1 决议） | 备选（React 线） | 选型理由 |
|---|---|---|---|
| 语言 | **TypeScript（strict: true 强制）** | 同 | 金融场景类型安全、契约驱动（DD-01） |
| 框架 | Vue 3 + `<script setup>` + Composition API | React 18 + 函数组件 + Hooks | **公司前端团队更熟悉 Vue（D1 决议选 Vue3 线）**；React 线须禁用 Options API 混用 |
| 构建 | **Vite 5** | 同 | 秒级冷启、原生 ESM、路由级 code-splitting |
| UI 库 | Element Plus / Ant Design Vue | Ant Design 5（组件）+ AntV 体系 | 金融中后台成熟、主题算法（暗/亮，Ant Design Vue 主题） |
| 拓扑 | **@antv/g6 v5（WebGL 渲染器，D2 推荐方案 A）** | 同 | 力导/分层、万级节点 WebGL、LOD/聚类 |
| 图表 | ECharts 5 / @antv/g2 | 同 | 趋势/仪表 |
| 火焰/瀑布 | 自研 Canvas + Web Worker | 同 | 海量 Span 建树卸载主线程 |
| 状态-客户端 | Pinia（轻全局，禁用 Options API 混写） | Zustand | 低样板、Vue 官方推荐 |
| 状态-服务端 | **Vue Query（TanStack Query，缓存/重取/实时）** | TanStack Query | 统一服务端状态、避免手写缓存 |
| 路由 | Vue Router v4（懒加载） | React Router v6 | 路由级分包 |
| 测试 | Vitest + Vue Test Utils + Playwright(E2E) | Vitest + RTL + Playwright | 单测+组件+E2E |
| 规范 | ESLint + Prettier + Stylelint + Husky + lint-staged | 同 | 卡点 |

- **框架专项检查（落地约束，D1 决议选定 Vue3 线）**：
  - **Vue3 线（选定）**：统一 `<script setup>` + Composition API；状态用 Pinia；禁止 Options API 与 Composition 混写；重型列表/图表用 `computed`/`shallowRef` + `v-memo` 防重渲染；禁止在模板/渲染函数内创建大对象。
  - React 线（备选冻结）：所有组件函数式 + Hooks；重型列表/图表用 `memo`/`useMemo` 防重渲染；禁止在 `render` 内创建对象。
- **决策项（已决议 D1）**：**选定 Vue3 线**（公司前端团队更熟悉 Vue 技术栈），React 线作为备选冻结；本表以 Vue3 为锁定基线。

## 3. 前端架构与目录结构（评审 P1-1）

**Feature-based 分层**（杜绝上帝文件）：

```
src/
├─ app/            # 入口、路由、Provider、全局布局
├─ features/       # 按业务域拆分（与 DD 模块对齐）
│  ├─ dashboard/   # 概览大屏
│  ├─ topology/    # 拓扑（DD-16）
│  ├─ traces/      # Trace 查询/瀑布/三域
│  ├─ apm/         # 火焰图/热点/健康
│  ├─ infra/       # 基建监控
│  ├─ alerts/      # 告警
│  ├─ admin/       # 管理后台
│  └─ ingest/      # 采集治理
├─ shared/         # 跨域复用：components/hooks/services/utils
│  ├─ components/  # MetricCard/TimeRangePicker/DataTable/...
│  ├─ hooks/       # useRealtime/usePermission/useTraceError
│  ├─ services/    # api 客户端（DD-01 契约）
│  └─ stores/      # Pinia 客户端状态（Vue3 选定）
├─ lib/            # Query 配置、RealtimeService、设计令牌
└─ types/          # 契约类型（DD-01 生成）
```

- **分层原则**：页面只组装，业务逻辑下沉 `features/*/hooks`；跨域组件进 `shared`；服务端状态走 Vue Query（TanStack Query，不进全局 store）；权限/错误/实时为横切 `hooks`。
- **多应用考量**：若大屏与后台差异大，建议 `pnpm workspace` + 共享 `@npm/ui-kit`（P2-4），当前单 SPA 即可。

## 7. 可视化渲染方案（评审性能专项）

| 图 | 技术选型 | 关键策略 |
|---|---|---|
| 拓扑图（DD-16） | @antv/g6 v5 **WebGL 渲染器（D2 推荐方案 A）** | **万级节点**：WebGL 渲染 + 视口 LOD + 按 scope 聚类收起（DD-16 scope=global/system/center/room）+ **Web Worker 计算布局** |
| 瀑布图 | 自研 Canvas + **虚拟化** | Worker 从 ≤1万 Span 建时序树；仅渲染可视时间窗；父子嵌套着色 |
| 火焰图 | 自研 Canvas + **Web Worker** | Worker 从 ≤5万 Span 聚合 `parent_span_id` 层级 + self_time；代表 Trace 聚合（DD-03 §9.2） |
| 趋势图 | ECharts 5 | 时间序列，P95/错误率叠加（按需分包） |
| 健康仪表 | SVG 仪表 | 0~100 分级（色+文字） |

- **性能总纲**：重计算进 Worker（不卡主线程）；重渲染用 Canvas + 虚拟化；重库（g6/echarts）路由级按需分包；客户端缓存降重绘（Vue Query staleTime）。
- **拓扑渲染评审决议（D2）**：前端渲染**推荐方案 A**——G6 v5 WebGL 渲染器 + 视口 LOD + 按 scope 聚类收起 + Web Worker 布局；**方案 B（默认聚类收起、仅下钻展开）作为降级兜底共存**（超阈值自动切聚类）。后端拓扑数据由 DD-16 重新聚合落库 ES，前端仅检索轻量聚合。

## 8. 性能优化专项（评审 P1-4 / 原待评审项 2、3）

### 8.1 万级拓扑节点（原待评审 2）
- **渲染**：G6 v5 WebGL 渲染器（GPU 绘制），避免 SVG/DOM 万节点卡死。
- **LOD**：视口外/缩小时降细节（隐藏标签、合并边），放大才显全。
- **聚类**：默认按 `scope`（中心/机房）收起子图，下钻展开（呼应 DD-16）。
- **布局**：力导/分层计算放 **Web Worker**，主线程只收坐标（不阻塞交互）。
- **数据**：拓扑快照走 Redis `topology:cache`(#3) + ES 轻量聚合（DD-02/16），前端不重算。

### 8.2 火焰图/瀑布图大数据（原待评审 3 延伸）
- 建树/聚合（5 万 Span）在 **Web Worker** 完成，结果 `postMessage` 传主线程渲染。
- 瀑布图 **虚拟化**：仅渲染当前时间窗可视 Span，滚动/缩放动态加载（DD-03 §9.4 展示上限 1 万）。

### 8.3 包体与首屏（原待评审 3）
- **路由级 `lazy()`** + Vite `manualChunks` 将 `g6`/`echarts` 拆独立 chunk，进拓扑/APM 页才加载。
- **量化目标（P2-3）**：首屏 FCP < 1.5s、LCP < 2.5s、主包 < 300KB(gzip)；接入 Lighthouse CI 卡点（DD-11 工程规范）。

## 9. 实时更新策略（评审 P1-2 / 原待评审 1）

> 原"WS 或轮询"未决，本版**定稿统一实时层**。

- **统一 RealtimeService**：基于 **WebSocket**，封装心跳（30s）+ 指数退避重连（200ms→上限 10s）+ 断线降级。
- **降级**：WS 不可达/代理不友好时降级 **SSE**；SSE 也不行则**轮询回退**（默认 15s，可配）。
- **按页面分级**：

| 页面 | 实时方式 | 时效性 |
|---|---|---|
| 概览大屏 / 告警 / 采集治理 | WS 增量推送 | 秒级 |
| 拓扑 | WS 增量 + 60s 快照合并（DD-16） | 秒~分 |
| APM / 基建监控 | Query 周期重取（15s）或 WS | 秒~15s |
| Trace 查询 | 按需拉取（无实时） | — |

- **客户端缓存**：Query `staleTime` 对齐后端刷新频率，避免重复请求；实时推送经 Query 的 `queryClient.setQueryData` 就地更新。

## 10. 前后端接口对接

- **契约**：严格遵循 DD-01 `/api/v1` REST；统一响应 `{code,message,data,traceId}`；Types 由契约生成（DD-11 提测标准）。
- **错误态**：拦截器统一处理错误码（§6），友好文案 + traceId（便于后端定位）。
- **权限态**：403 时按 `perm:cache`(DD-09) 隐藏/置灰，真实鉴权在服务端（DD-09 二次校验）。
- **Mock 约定**：并行期按 DD-01 字段契约出 Mock（MSW/本地），稳定后切真实。

## 12. 安全与合规（前端侧）

- **XSS**：模板自动转义 + 动态内容显式转义；CSP 策略；富文本白名单（DD-09 §6）。
- **敏感信息**：令牌存 HttpOnly Cookie/内存，日志不打印敏感（DD-14）。
- **越权防护**：前端隐藏仅体验，服务端二次校验（DD-09）。

## 13. 可访问性与主题（评审 P2-1/P2-2 / 原待评审 4）

- **可访问性（WCAG 2.1 AA）**：
  - **色盲安全配色**：状态/严重级用"色 + 图标 + 文字"三重标识（如 健康✓绿 / 警告⚠黄 / 严重✕红），不单靠颜色（Okabe-Ito 安全色板）。
  - **键盘可达**：拓扑节点可 Tab/方向键聚焦、表格可键盘导航、`focus-visible` 轮廓。
  - **对比度**：暗色大屏文本对比 ≥ 4.5:1；尊重 `prefers-reduced-motion` 关闭非必要动画。
- **主题（设计令牌）**：CSS 变量 + 设计令牌；两套算法——**暗色（大屏/概览）**与**亮色（管理后台）**，经 Ant Design 主题算法切换，全站令牌统一（色/距/圆角/字号）。

## 14. 设计原则遵循声明

- 性能✅（Worker 卸载、Canvas 虚拟化、WebGL 拓扑、viz 分包、Query 缓存、P95<1s 体感） 可观测性✅（状态/错误/延迟全可视、响应带 traceId） 可维护性✅（Feature 分层、组件化、契约驱动、工程化卡点） 零信任✅（权限态/错误态、服务端鉴权） 防御式✅（XSS/CSP、错误拦截） 资源消耗红线✅（前端轻量、分包控主包） 可访问性✅（WCAG AA、色盲配色、键盘导航）。
- **原 4 待评审项处理（已给推荐方案）**：
  1. 实时更新选型 → 定稿 **WS 统一层 + SSE 降级 + 轮询回退 + Query 缓存**，按页面分级（§9）。
  2. 万级拓扑渲染 → **G6 WebGL + LOD + 聚类 + Worker 布局**（§8.1）。
  3. 包体/首屏目标 → **路由 lazy + viz 分包 + Lighthouse CI**，量化 FCP/LCP/主包（§8.3）。
  4. 暗色大屏/a11y → **设计令牌双主题 + WCAG AA + 色盲配色 + 键盘导航**（§13）。
- **评审会决议（已落地，见决议清单 D1/D2）**：
  - **D1 前端框架 = Vue3 线**（公司前端团队更熟悉 Vue 技术栈）：§1 已锁定 Vue3 基线，React 作备选；框架/UI 库/状态/路由/测试/主题全部按 Vue3 实现。
  - **D2 拓扑前端渲染 = 推荐方案 A**（G6 v5 WebGL + 视口 LOD + 按 scope 聚类收起 + Web Worker 布局），B 聚类优先作降级兜底；后端拓扑数据见 DD-16（重新聚合落库 ES）。
- **仍待决策/验证（非 D1–D13 决议项）**：拓扑 WebGL PoC 验证万节点帧率与 Worker 布局耗时（P0-2）；Lighthouse 具体阈值（P2-3）。
