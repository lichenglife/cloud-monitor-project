## 1. 安全总体框架（零信任）

> 默认不信任任何内外部请求，全部鉴权；权限以 **CMDB 兜底最小权限（决议 D）** 为准。

| 层 | 安全措施 |
|---|---|
| 接入层 | API 网关统一鉴权/限流/审计（DD-01）；mTLS 全链路 |
| 采集层 | Agent 注册令牌 + TLS；Collector 仅接受鉴权 Agent（DD-07/DD-13） |
| 数据层 | 敏感配置 AES-256-GCM 加密；PII 源头脱敏（DD-14）；传输 TLS |
| 权限层 | CMDB 数据权限兜底 + 平台 RBAC（决议 D） |
| 审计层 | 全操作审计 `t_audit_log`（DD-02） |

## 2. 身份认证

- **用户认证**：`user-auth` 服务 `VerifyToken`（DD-01 §5）；会话 `user:session:{token}`(Redis #7，TTL 2h)；令牌定期刷新。
- **Agent 认证**：注册握手签发 `t_agent_registration.token`，上报携带鉴权（DD-13 §7）；令牌加密存储（§4）。
- **服务间**：内部 gRPC mTLS（Collector/聚合/告警等），零信任内部调用。

## 3. 权限模型（RBAC + CMDB 兜底，决议 D）

```
用户(t_user) —< 角色(t_role) —< 权限(t_permission)
                        │
                        ▼
               CMDB 数据权限（应用/系统/机房/中心）
                        │
                        ▼
           perm:cache:{userId}(Redis #12，TTL 5m) → 最小权限兜底
```

- **平台 RBAC**：用户/角色/权限表（DD-02 §1.2）；功能权限（菜单/接口）由角色控制。
- **数据权限（CMDB 兜底）**：可见应用/系统/机房/中心范围由 CMDB 同步；平台无显式授权时**默认拒绝**（最小权限），避免越权查看其他团队 Trace。
- **缓存与失效**：`perm:cache:{userId}`(Redis #12) 缓存权限，CMDB/角色变更时主动失效（≤5m 自动过期兜底）。
- **拓扑/链路可见性**：DD-16 拓扑、DD-03 各模块查询均按 CMDB 权限过滤（无权限节点/服务不返回或置灰）。

## 4. 敏感数据加密

| 数据 | 加密方式 | 存储 |
|---|---|---|
| Agent 注册令牌 | AES-256-GCM | `t_agent_registration.token`（DD-02） |
| 用户凭证/密钥 | AES-256-GCM + 盐 | `t_user` 等 |
| 配置中心密钥 | KMS 托管 + AES-256-GCM | config-center |
| 传输中 | TLS 1.2+/mTLS | 全链路 |

- 密钥管理：KMS 统一托管主密钥，定期轮换；明文绝不落日志/ES（DD-14 脱敏协同）。

## 5. 数据脱敏与防泄漏（合规红线）

- **源头脱敏**：采集层（Agent/Collector Transform）对 PII（手机号/身份证/密码/银行卡）掩码（DD-14 §4.2），敏感数据不落盘/不外传。
- **查询脱敏**：即便原始 `attributes`(flattened) 留存，高基数敏感字段不索引、展示侧再掩码兜底。
- **日志脱敏**：日志平台关联（DD-03 §7）同样遵循，避免 TraceID 关联泄漏。
- **PII 泄漏扫描**：数据质量监控含 PII 泄漏扫描（DD-14 §6），命中即告警（DD-06）。

## 6. 应用安全防护

| 风险 | 防护 |
|---|---|
| XSS | 前端（DD-15）输出编码、CSP；富文本白名单 |
| SQL 注入 | 统一 ORM/参数化；DD-02 查询经 Repository 抽象 |
| 文件上传 | 类型/大小校验、隔离存储、病毒扫描（管理后台导入） |
| 越权 | CMDB 兜底最小权限 + 服务端二次校验（不依赖前端） |
| 限流滥用 | API 网关 2000 QPS 集群 / 100 单用户（DD-01 §9） |

## 7. 审计与合规

- 全关键操作（配置变更、权限调整、Agent 升级、数据导出）写入 `t_audit_log`（operator+action+time，DD-02 索引）。
- 审计日志自身不可篡改（append-only，定期归档）；合规报表支持按时间/操作人追溯。

## 8. 设计原则遵循声明
