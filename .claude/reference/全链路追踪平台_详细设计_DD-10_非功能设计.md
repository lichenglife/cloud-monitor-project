## 1. 性能设计（目标 P95<1s）

| 场景 | 指标 | 目标 | 保障手段 |
|---|---|---|---|
| Trace 查询（DD-03 §4） | 列表/详情响应 | P95 < 1s | ES trace 索引优化、预聚合、多级缓存 |
| 拓扑/概览（DD-16/§3） | 看板响应 | P95 < 1s | Redis 拓扑缓存(#3)、ES 内聚合 |
| APM 火焰图/健康（§5） | 渲染响应 | P95 < 1s | 预聚合 + 火焰图缓存（§9.2） |
| 告警评估（DD-06） | 规则评估 | 实时（秒级） | 基于预聚合索引、无重聚合 |
| 写入（采集→Kafka→ES） | 摄入延迟 | P95 < 2s | 无状态 Collector、Kafka 高吞吐、背压 |

- **查询性能**：聚合结果预计算落库（DD-02 6 索引），查询仅轻量检索/简单 term 聚合；计算贴近存储（ES 内聚合）；热点走 Redis 缓存（多级缓存 Caffeine+Redis）。
- **写入性能**：Collector 全异步非阻塞（Netty/Reactor）；批量导出 Kafka；ES Ingest Pipeline/Transform 聚合（DD-02 定稿）。

## 2. 高可用与容灾（Failover）

| 组件 | HA 方案 | 故障影响 |
|---|---|---|
| Collector | 多实例 + K8s HPA + 本地磁盘缓冲 | 单点故障无感切换，数据不丢 |
| Kafka | RF≥3 + 跨 AZ 副本（上海 DC1，决议 A） | 单节点故障不丢 |
| ES | 多节点 + 副本 1 + 分片水平扩 | 单节点故障可读副本 |
| Redis | Cluster 分片 + 主从 + AOF | 单分片故障切换 |
| MySQL | 主从 + 读写分离 + 每日全量+binlog | 主库故障切从 |
| 平台服务 | 多副本 + `/health` 探活 + 无状态 | 滚动/金丝雀发布无中断 |

- **降级优先级**：保采集 > 保聚合展示（DD-07 §5）；ES 不可用时 Collector 缓冲，不反向压垮业务（采集链路与存储链路解耦）。
- **多中心容灾**：详见 DD-08（各中心 Collector 独立、上海 DC1 集中、异地异步备份待二期）。

## 3. 容量规划（试点基线，呼应 DD-04）

| 资源 | 试点规模 | 测算 | 备注 |
|---|---|---|---|
| ES | 300 工程 / 峰值 500 QPS / 1 年 | trace ~315TB（压缩后），聚合索引 ~数 TB / 180d | ILM 热→冷（DD-04） |
| Kafka | 峰值 ×3 安全系数 | 分区 12(span)/6(metric)/6(log)/3(其他) | 按 trace_id hash 保序 |
| Redis | 热配置/缓存/会话 | 数十 GB（allkeys-lru，热配置常驻） | 单 Value<10KB |
| MySQL | 元数据为主 | 单表 >2000 万行 / 50GB 触发分表 | Z 轴分表（DD-02） |
| 计算 | Collector/聚合/告警 | HPA 按 CPU/吞吐 | 无状态 |

- **压测方案（评审决议 D10=方案A）**：**试点期先按 3× 流量压测**验证背压/降级/不丢（设计心法 Scale），**上线前按 10× 全量压测**出具压测报告作为 Gate2.5 附件；验证 Collector 吞吐、ES 检索、Kafka Lag。

## 4. 限流与熔断

- **API 网关**：集群 2000 QPS / 单用户 100 QPS（DD-01 §9）；超阈 429 退避。
- **Collector OTLP 入口**：试点 1500 QPS 预留，超阈背压（DD-07 §5）。
- **服务间**：gRPC 超时 1000ms + 幂等重试 2 次（DD-01 §9）；非幂等不重试。
- **熔断**：聚合/ES 异常时熔断降级（返回缓存/降级视图），不雪崩。

## 5. 可观测性（Dogfooding，呼应 DD-05）

- 平台自身黄金指标（Collector/Kafka/ES/Redis/MySQL 的 RED/USE）经本平台采集展示（吃自己狗粮）。
- 平台健康看板（DD-15）+ `/health` 探活 + 自监控告警（DD-06 §7）。

## 6. 设计原则遵循声明

- 性能✅（P95<1s、预聚合、多级缓存、异步） 可靠性✅（多副本、降级、背压、不丢） 伸缩性✅（分片、HPA、分表） 成本✅（采样降本、ILM 降采样、压缩） 可观测性✅（Dogfooding 黄金指标） 资源消耗红线✅（Agent CPU<1%/内存<128M，DD-13）。
