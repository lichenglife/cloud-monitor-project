## 4. Collector 上报协议（OTLP）
- **OpenTelemetry Protocol（OTLP）**：gRPC `:4317`、HTTP `:4318`，Payload 为 `ExportTraceServiceRequest` / `ExportMetricsServiceRequest` / `ExportLogsServiceRequest`。
- Agent 启动参数注入 Collector 地址（biz 直连 / dmz 代理转发，决议 A）；各中心 Collector 汇聚后统一上报上海 DC1 Kafka（决议 A）。
- **幂等与背压**：Collector 按 `trace.id+span.id` 去重；ES 不可用时降级缓冲至本地磁盘队列，不反向压垮 Agent（可靠性原则）。

## 5. 内部服务 gRPC 接口（14 服务，节选）
| 服务 | 核心 RPC | 说明 |
|---|---|---|
| collector-gateway | `PushSpan`/`PushMetric`/`PushLog` | 接收 OTLP 并转发 |
| agent-mgr | `GetAgentConfig`/`PushAgentHeartbeat`/`PushUpgrade` | Agent 配置/心跳/升级 |
| trace-store | `SaveTrace`/`QueryTrace` | ES 读写（统一 Repository 接口） |
| topology-calc | `CalcTopology` | 拓扑计算（独立服务，SRP） |
| alert-engine | `Evaluate`/`EmitAlert` | 告警评估与投递 |
| config-center | `Watch`/`Publish` | 配置热更新（Watch 推送 ≤30s） |
| user-auth | `VerifyToken`/`CheckPerm` | CMDB 权限校验（最小权限兜底） |
| api-gateway | 聚合编排 | 对外 REST 网关 |

> 存储访问经**统一 Repository 抽象**（LSP/DIP）：ES、MySQL 适配实现同一接口。

## 7. 错误码规范
| code | 含义 | 处理 |
|---|---|---|
| 0 | 成功 | — |
| 40001 | 参数校验失败 | 防御式校验，字段级错误 |
| 40101 | 未认证 | 跳转鉴权 |
| 40301 | 无权限（CMDB 兜底最小权限） | 拒绝并记审计 |
| 40901 | 幂等冲突 | 幂等键去重 |
| 42901 | 限流 | 客户端退避重试 |
| 50001 | 内部错误 | 返回 traceId 供定位 |

## 8. 版本化与向后兼容
- REST 路径含 `/v1`；Kafka 消息 **Avro + Schema Registry**，新增字段兼容、删改字段升 `v2`。
- 接口废弃走 Deprecation 周期，不强行下线（向上兼容）。

## 9. 设计原则遵循声明
- 向后兼容✅（版本化/Schema Registry） 幂等性✅（Kafka/API 去重） 无状态✅（网关/Collector） 零信任✅（全链路鉴权） 防御式编程✅（参数/返回值校验） 可观测性✅（响应带 traceId） SRP✅（topology-calc 独立） LSP/DIP✅（统一 Repository）
- **定稿决议（原待评审项已按推荐方案落定）**：
  - **gRPC 服务间超时/重试**：默认超时 **1000ms**；仅幂等 RPC 重试 **2 次**、指数退避（200ms→400ms）；非幂等写操作不重试。
  - **限流阈值**：API 网关集群级 **2000 QPS**、单用户 **100 QPS**（超阈 429 退避）；Collector OTLP 入口按试点峰值 QPS 500 × 安全系数 3 ≈ **1500 QPS** 预留，按 Agent 数自动评估扩容。
  - **接口字段全集**：本版冻结接口契约骨架（方法/路径/页面归属/关键参数）；字段级 request/response 明细在 **DD-03 模块详细设计**中按模块深化，不阻断定稿。
- **定稿状态**：v1.0 已定稿，作为编码与前端（DD-15）契约基线。
