## Context

M4 拓扑已就绪（npm_online_topology 边 + 快照）。M5 在 topology-service 上扩展影响面计算，复用拓扑数据，不重建。设计见 DD-16 §3-4 + DD-03 §3。

## Goals / Non-Goals

**Goals:**
- 上下游影响面计算（depth=1/2/3）
- 爆炸半径输出（受影响服务/实例/层级，强弱依赖区分）
- 拓扑盲区提示
- 告警联动入口

**Non-Goals:**
- 不重建拓扑（复用 M4）
- 不实现告警规则引擎（M5.1）
- 不做深度 RCA（M10，二期）

## Decisions

### DM1: 沿拓扑 BFS 计算影响面
故障服务为起点，沿 npm_online_topology 边 BFS 遍历上游（反向边）与下游（正向边），depth 可配（1/2/3）。强依赖边传播"阻断"影响，弱依赖边标注"可降级"。

### DM2: 爆炸半径聚合
聚合影响面为 {受影响服务数, 实例数, 最大层级深度, 强依赖影响数, 弱依赖影响数}，前端可视化（影响面子图 + 统计卡）。

### DM3: 拓扑盲区诚实标注
拓扑不完整（未知依赖/缺失边）时，影响面标注"可能存在遗漏"，不绝对结论（PRD REQ-M5-W1）。

## Risks / Trade-offs

- **拓扑不全致影响面遗漏**：缓解--盲区提示 + 持续提升拓扑自动发现覆盖率（M4 ≥90%）。
- **大拓扑 BFS 性能**：缓解--depth 上限 3 + 快照缓存（Redis topology:upstream:{service}:{depth} 300s TTL）。
