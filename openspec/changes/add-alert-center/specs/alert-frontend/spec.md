## ADDED Requirements

### Requirement: 告警中心页
前端 MUST 实现告警中心页（规则管理 CRUD/启停、事件列表按级别/时间/服务/状态筛选、静默配置、Oncall 值班展示、告警详情 + 关联 Trace 按钮 + 降噪原因标注），MUST 支持实时 WS 增量推送告警事件。

#### Scenario: 告警实时推送
- **WHEN** 新告警事件触发
- **THEN** 前端经 WS 增量推送实时显示，无需手动刷新

#### Scenario: 告警一键进入链路
- **WHEN** 用户点击告警的"关联 Trace"按钮
- **THEN** 跳转链路详情页（≤3 次点击进入链路，PRD US-01）

### Requirement: 状态覆盖与降级
MUST 覆盖加载/空/异常/无权限/降级态，统一告警平台不可达时标注"投递通道暂不可用"不影响事件落库。

#### Scenario: 投递通道不可用降级
- **WHEN** 统一告警平台不可达
- **THEN** 事件仍落库，标注"投递通道暂不可用"，恢复后补投
